## 摘要
序列推荐系统通常通过单次编码交互历史来推断用户偏好，而不进行迭代精化，依赖日益加深的架构来捕捉复杂模式。本文从递归推理的角度重新审视序列推荐：用户偏好能否建模为持续递归精化的潜在状态？我们提出RecRec（递归推荐），一种轻量级模型，它维护一个紧凑的潜在状态，并通过基于交互证据的共享递归模块进行更新。与先前的递归模型不同，RecRec引入了一种证据锚定的校正机制，通过将每次更新锚定在原始交互上下文中来稳定精化过程，防止深层递归推理中的语义漂移。在三个基准数据集上的标准评估协议实验表明，RecRec在仅使用3.9M至14M参数的情况下，匹配或超越了最先进的序列、基于图以及推理增强的推荐器。消融研究表明，递归精化和证据锚定的校正门机制对性能均有显著贡献，凸显了递归潜在推理作为更深层或基于语言的架构的可扩展替代方案的有效性。代码可在 https://anonymous.4open.science/r/RecRec-6B67/README.md 获取。

## Abstract
Sequential recommender systems typically infer user preferences through single-pass encoding of interaction histories without iterative refinement, relying on increasingly deep architectures to capture complex patterns. In this work, we revisit sequential recommendation from a recursive inference perspective: can user preferences be modeled as a persistent latent state that is recursively refined? We propose RecRec (Recursive Recommendation), a lightweight model that maintains a compact latent state and updates it through a shared recursive module conditioned on interaction evidence. Unlike prior recursive models, RecRec introduces an evidence-anchored correction mechanism that stabilizes refinement by grounding each update in the original interaction context, preventing semantic drift during deep recursive reasoning. Experiments on three benchmark datasets under standard evaluation protocols show that RecRec matches or outperforms state-of-the-art sequential, graph-based, and reasoning-enhanced recommenders while using only 3.9M to 14M parameters. Ablation studies demonstrate that both recursive refinement and the evidence-anchored correction gate contribute significantly to performance, highlighting the effectiveness of recursive latent inference as a scalable alternative to deeper or language-based architectures. Code is available at https://anonymous.4open.science/r/RecRec-6B67/README.md.

- **现有序列推荐系统的局限**：主流方法（如马尔可夫模型、RNN、Transformer）通常一次性编码用户交互历史，缺乏迭代修正机制，依赖深层网络来捕捉复杂模式。这导致两个问题：一是无法逐步改进偏好估计或从近似误差中恢复；二是深层架构容易过拟合，且无法显式建模结构化推理。
- **大型语言模型（LLM）的瓶颈**：虽然LLM/SLM能够进行多步推理，但计算开销巨大（例如7B参数），且依赖语言模型的token空间，并非专门为推荐优化。
- **核心问题**：能否将用户偏好建模为一个持久的潜在状态，通过递归推理逐步精化，从而在轻量级参数下达到甚至超越深层或基于语言模型的方法？

## 2. 方法论：核心思想、关键技术细节

**核心思想**：  
RecRec 维持一个紧凑的潜在状态（preference state），通过共享递归模块基于交互证据迭代更新，并引入证据锚定校正门（evidence-anchored correction gate）来防止语义漂移。

**关键技术细节**（文字描述公式与流程）：

- **状态定义**：模型维护三个核心状态：
  - **静态上下文 x**：通过掩码平均池化从用户历史交互的SBERT嵌入（或随机嵌入）得到，作为初始偏好状态。
  - **偏好状态 y_t**：当前步骤的用户偏好表示，初始化为 x。
  - **潜在状态 z_t**：内部推理表示，初始化为零向量。

- **内循环（Inner Recursion）**：在每个外步骤 t 内，执行 n 次内递归更新（用共享非线性变换 f_φ）：
  \[
  z_{t}^{(j)} = f_{\phi}([x \| y_t \| z_{t}^{(j-1)}])
  \]
  其中 [·∥·] 为拼接，j 从1到n，z_t^{(0)} 继承自上一外步骤。内循环在固定偏好状态下精化潜在状态。

- **校正门（Correction Gate）**：计算门控值 g_t，用于在最终潜在状态与原始上下文 x 之间插值，防止漂移：
  \[
  g_t = \sigma(W_t [x \| y_t]), \quad \hat{z}_t = (1 - g_t) z_t^{(n)} + g_t x
  \]
  g_t 可视为对递归更新修正程度的自适应信任系数。

- **偏好精化（Preference Refinement）**：利用校正后的潜在状态更新偏好状态：
  \[
  y_{t+1} = y_t + L \cdot \tanh\left( f_{\phi}([x \| y_t \| \hat{z}_t]) \right)
  \]
  其中 L 为缩放因子，实现残差更新。

- **深度监督与损失**：总共 T 个外步骤，每个步骤产生的偏好状态 y_{t+1} 都通过交叉熵损失监督目标物品，总损失为所有步骤损失的平均。梯度通过内循环和外部更新反向传播。

**算法流程**（文字描述）：
1. 初始化 x (上下文)，y_0 = x，z_0 = 0。
2. 对于 t = 0,..., T-1：
   - 执行 n 次内递归更新 z_t（基于 x, y_t, 当前 z）。
   - 计算校正门 g_t 并得到校正后的潜在状态 ẑ_t。
   - 更新偏好状态 y_{t+1} = y_t + Δ_t。
   - 计算损失 L_CE(y_{t+1}, target) 并累加。
3. 使用平均损失进行反向传播。

## 6. 主要结论与发现

- RecRec 在三个数据集上以**仅3.9M-14M参数**（比LLM小99%以上）匹配或超越所有基线，尤其在前排名指标HR@1、NDCG@1上优势明显。
- **递归精化有效**：增加递归步数T从1到7，HR@1从0.49提升至0.59；超过7步后性能退化（过精化）。
- **证据锚定校正门至关重要**：移除校正门导致最大性能下降（表2中HR@1从0.59降至0.46）。
- **潜在状态和内循环均不可或缺**：两者移除均显著降低性能。
- **增益不依赖外部语义先验**：随机嵌入的RecRec-Random性能与SBERT嵌入版本相当（±2%），说明递归结构而非预训练语义驱动提升。
- 递归精化提供比单纯增加网络深度更参数高效的替代方案（5层MLP最优，10层反而过拟合）。

## 7. 优点

- **极轻量级**：3.9M-14M参数即可达到与7B LLM相当或更好的推荐性能，训练时间缩短98%+，可实际部署。
- **结构新颖**：首次将递归推理与证据锚定校正结合用于序列推荐，解决了递归模型中的语义漂移问题。
- **深度监督与共享参数**：通过每步损失监督和共享递归模块，实现稳定优化与知识复用。
- **消融实验全面**：涵盖核心组件、步数、嵌入类型、模型深度等，验证了设计合理性。
- **跨域泛化**：在三个不同领域（美妆、游戏、软件）均表现优异。

## 8. 不足与局限

- **对比方法不够全面**：未与当前最新的基于Transformer的大规模序列模型（如BERT4Rec、CL4SRec）或基于图注意力网络的方法进行实验。
