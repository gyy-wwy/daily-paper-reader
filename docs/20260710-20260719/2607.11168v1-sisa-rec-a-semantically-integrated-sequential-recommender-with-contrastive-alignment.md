## 摘要
推荐系统帮助用户从大量候选项中选择相关物品。当前基于Transformer的序列推荐研究从交互日志中学习用户偏好，但主要关注物品标识符，未能充分利用物品的语义含义。这一局限在历史交互数据有限的稀疏和冷启动场景中成为重大挑战。为解决此问题，我们提出了SISA-Rec（语义集成序列推荐），一种将语义上下文直接嵌入序列建模的Transformer框架。该方法通过门控融合模块融合物品ID嵌入与基于BERT的文本嵌入，将语义相似性注入自注意力机制，并利用基于注意力的聚合模块构建全面的用户表示。最后，结合贝叶斯个性化排序（BPR）和对比对齐损失的联合学习目标，对齐底层行为空间和语义空间。在高度稀疏的Amazon Beauty和Amazon Toys & Games两个数据集上进行实验，两者稀疏度均达到99.93%。结果表明，SISA-Rec在所有评估指标上均优于最先进的基线模型。与BERT4Rec相比，SISA-Rec在Amazon Beauty上HR@10提升16.6%，NDCG@10提升10.3%；在Amazon Toys & Games上HR@10提升23.1%，NDCG@10提升17.9%。冷启动分析进一步表明，该模型对历史交互记录有限的用户提升最大，这展示了在用户行为数据稀缺时语义信息的价值。总体而言，结果表明将语义信息集成到注意力机制中能够产生更准确可靠的推荐。

## Abstract
Recommendation systems help users recommend relevant items from a large collection of choices. Present work on transformer-based sequential recommendation learns user preferences from interaction logs, but it mostly focuses on item identifiers and doesn't fully use the semantic meaning of items. This limitation becomes a major challenge in sparse and cold-start scenarios where historical interaction data is limited. To solve this problem, we introduce SISA-Rec (Semantically Integrated Sequential Recommendation), a transformer-based framework that embeds semantic context directly into sequential modeling. Our approach fuses item ID embeddings with BERT-based text embeddings via a gated fusion module, injects semantic similarity into the self-attention mechanism, and leverages an attention-based aggregation module to construct comprehensive user representations. Finally, a joint learning objective which combines Bayesian Personalized Ranking (BPR) and contrastive alignment loss, aligns the underlying behavioral and semantic spaces. Experiments were conducted on the two highly sparse Amazon Beauty and Amazon Toys \& Games datasets, both having 99.93\% sparsity. The results show that SISA-Rec outperforms state-of-the-art baseline models across all evaluation metrics. Compared with the BERT4Rec \cite{petrov2022systematic}, SISA-Rec improves HR@10 by 16.6\% and NDCG@10 by 10.3\% on Amazon Beauty, and HR@10 by 23.1\% and NDCG@10 by 17.9\% on Amazon Toys \& Games. Cold-start analysis further shows that the proposed model achieves the largest improvements for users with limited interaction historical records. This showcases the value of semantic information when user behavior data is scarce. Overall, the results demonstrate that integrating semantic information into the attention mechanism leads to more accurate and reliable recommendations.

- **动机**：虽然已有一些语义感知模型（如UniSRec、MoRec）引入了物品文本表示，但它们通常将语义作为“副信号”在序列编码之后才进行融合，或者以独立模块附加在现有架构之上，导致语义没有真正参与序列建模的核心过程（即自注意力计算）。这限制了语义对建模过程的直接影响。
- **论文目标**：设计一个框架，使得语义表示从输入层开始就活跃在编码的每一个阶段，特别是直接注入自注意力机制，从而在行为信号稀少时利用语义关系提供补充信息。

## 2. 提出的方法论

### 核心思想
SISA-Rec 是一个融合了物品ID嵌入和BERT语义嵌入的Transformer序列推荐框架。创新点包括：**门控融合模块**、**语义增强的自注意力**、**注意力偏好聚合**、**联合训练目标**（BPR损失 + 对比对齐损失）。

### 关键技术细节
1. **门控融合模块**  
   - 对每个物品，分别获取ID嵌入 \(e^{id} \in \mathbb{R}^d\) 和经过线性投影的BERT语义嵌入 \(e^{sem} \in \mathbb{R}^d\)。  
   - 计算门控向量 \(g = \sigma(W_g[e^{id}; e^{sem}] + b_g)\)，其中 \(\sigma\) 是sigmoid，\([;]\) 是拼接。  
   - 最终物品表示为 \(v = g \odot e^{id} + (1-g) \odot e^{sem}\)，门控控制每个维度上ID和语义的权重。

2. **语义增强的自注意力**  
   - 在标准自注意力中，注意力分数计算通常为 \(QK^T/\sqrt{d}\)，在此基础上加上语义相似度矩阵 \(S\)（由BERT嵌入的余弦相似度构建），即 \(A = \text{softmax}\left(\frac{QK^T}{\sqrt{d}} + \alpha S\right)V\)，其中 \(\alpha\) 是可学习的缩放参数。
   - 这允许语义相关的物品（即使从未在历史中共现）在注意力中获得更高的权重。

3. **注意力偏好聚合模块**  
   - 经过Transformer编码后，不直接使用最后一个隐藏状态作为用户表示，而是计算所有位置（非填充位）的注意力权重 \(a_t = w^\top h_t\)，经softmax归一化得到\(\alpha_t\)，然后加权求和得到用户偏好向量 \(u = \sum_t \alpha_t h_t\)。

4. **联合训练目标**  
   - **BPR损失**：\(L_{BPR} = -\frac{1}{|U|} \sum_u \log \sigma(s_{i^+} - s_{i^-})\)，鼓励正样本得分高于负样本。
   - **对比对齐损失**：基于InfoNCE，使行为偏好向量 \(u\) 与同一用户的语义偏好向量 \(u^{sem}\)（由用户历史物品BERT嵌入平均得到）相似，与不同用户的语义表示相异：\(L_{CL} = -\log \frac{\exp(u\cdot u^{sem}/\tau)}{\sum_j \exp(u\cdot u_j^{sem}/\tau)}\)。
   - 总损失 \(L = L_{BPR} + \lambda_1 L_{CL} + \lambda_2 \|\Theta\|^2\)，其中 \(\lambda_1=0.1, \lambda_2=1\times10^{-4}\)。

### 算法流程简述
1. 数据预处理：过滤低频用户（≥5次交互），按时间排序，留一法划分训练/验证/测试。
2. 文本拼接（标题+类别），用冻结的BERT提取768维语义向量，投影到d维。
3. 构建门控融合后的物品嵌入，加上位置编码。
4. 通过N层Transformer（修改后自注意力）编码。
5. 使用注意力聚合得到用户向量 \(u\)。
6. 用 \(u\) 与候选物品融合表示内积计算得分，取Top-N推荐。
7. 联合优化BPR和对比对齐损失。
