## 摘要
在异质二分图上训练的图神经网络构成了推荐系统的基础。这些图通常表达基数不同的关系，例如，用户-物品偏好是一对多的，而用户-属性特征是一对一的。传统上，所有网络组件使用统一的损失函数，通常是贝叶斯个性化排序（BPR）。虽然BPR在推荐任务中表现良好，但我们发现它会导致属性嵌入坍缩为近似随机的几何结构——这是一种隐性故障，标准排序指标基本不受影响，因此常规评估无法察觉。这继而污染了用户节点嵌入（用户节点嵌入同时由两种边类型塑造），损害了下游任务如个性化、分群等。本文提出了一种基数分解损失（CDL），结合了交叉熵（CE）和BPR，使模型能够跨基数关系进行联合优化。我们通过展示两个损失在共享编码器参数空间中竞争，证实了这种CE-BPR冲突。我们在五个数据集上评估CDL，涵盖两种结构配置——用户节点上的一对一属性（MovieLens-1M、Last.fm-360K、PayPal Audience Factory、BookCrossing）和物品节点上的属性（Yelp），发现CDL一致地改善了属性嵌入的可区分性。我们还表明，当属性携带有效的偏好信号时，排序（NDCG）得到提升，但当相关性较弱时则与之冲突。我们使用lambda参数来权衡这一折中，lambda扫描揭示数据集行为由两个图属性控制：语义对齐和拓扑泄露。语义对齐衡量属性是否预测偏好，而拓扑泄露衡量图的连通性是否已经编码了偏好。

## Abstract
Graph Neural Networks trained on heterogenous bipartite graphs form a common basis in recommendation systems. These graphs often express relations that vary in cardinality, for example, user-item preferences are one-to-many and user-attribute features are one-to-one. Traditionally, a unique loss function is applied for all of the network components which is often Bayesian Personalized Ranking (BPR). While BPR works well for the recommendation task, we find that it causes attribute embeddings to collapse to near-random geometry -- a silent failure that leaves standard ranking metrics largely unaffected and therefore invisible to conventional evaluation. This in turn pollutes user node embeddings, which are shaped by both edge types simultaneously, hurting downstream tasks like personalization, segmentation, etc. Here we propose a Cardinality-Decomposed Loss (CDL) that combines both Cross Entropy (CE) and BPR to enable the model to collectively optimize for relations across cardinalities. We confirm this CE-BPR conflict by showing the two losses compete in the shared encoder's parameter space. We evaluate CDL on five datasets spanning two structural configurations -- one-to-one attributes on user nodes (MovieLens-1M, Last.fm-360K, PayPal Audience Factory, BookCrossing) and on item nodes (Yelp) -- and find that CDL consistently improves discriminability in attribute embeddings. We also show that ranking (NDCG) improves when attributes carry meaningful preference signal, but conflicts with it when the correlation is weak. We use a lambda parameter to navigate this trade-off, and a lambda-sweep reveals that dataset behavior is governed by two graph properties -- semantic alignment and topology leakage. Semantic alignment measures whether the attribute predicts preferences, while topology leakage measures whether the graph's connectivity already encodes it.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：在基于图神经网络（GNN）的推荐系统中，输入图通常是异质的，包含用户-物品交互边（一对多关系）和用户属性边（如性别、年龄、职业等一对一关系）。现有方法对所有关系类型统一使用贝叶斯个性化排序（BPR）损失。
- **问题**：作者发现，在一对一属性上使用BPR损失会导致属性嵌入塌缩为近似随机几何结构（均值余弦相似度高达0.93-0.98），而标准排序指标（NDCG@10、Hit@10）几乎不受影响，因此这种故障是“静默的”，无法被常规评估察觉。这种塌缩同时污染了用户嵌入，损害下游任务（如个性化、分群）。
- **核心动机**：需要一种损失函数，能够根据关系结构（基数）匹配训练目标，避免属性嵌入退化，并提升整体模型质量。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出Cardinality-Decomposed Loss（CDL），根据关系基数分配不同的损失函数——对一对多（OTM）关系使用BPR，对一对一（OTO）关系使用交叉熵（CE），并将两者按加权和结合。
- **关键技术细节**：
  - 关系基数映射κ：从图结构统计中确定（源节点到目标节点度数为1且目标节点集小且完全枚举）。
  - BPR损失：用于用户-物品等一对多关系，标准形式：
    \[
    L_{\text{BPR}} = -\mathbb{E}_{(u,v^+,v^-)} \left[ \log\sigma(e_u^\top e_{v^+} - e_u^\top e_{v^-}) \right]
    \]
  - CE损失：用于用户-属性等一对一关系，为每个OTO关系附加线性分类头，对分类节点计算交叉熵：
    \[
    L_{\text{CE}} = -\mathbb{E}_u \left[ \log\frac{\exp(w_{y_u}^\top e_u)}{\sum_{k=1}^K \exp(w_k^\top e_u)} \right]
    \]
  - 总损失：\(L_{\text{CDL}} = L_{\text{BPR}} + \lambda \cdot L_{\text{CE}}\)，λ是唯一新超参数。
- **基线**：对比“otm_only”（只对OTM边使用BPR，OTO边仅用于消息传递无监督，视为CF基线）和“uniform”（所有边使用BPR，视为错误基准）两种模式。
- **梯度冲突分析**：测量共享编码器上两个损失梯度的余弦相似度，证实两者持续为负，表明BPR和CE在共享参数上竞争。

## 6. 主要结论与发现

1. **统一BPR导致属性嵌入静默塌缩**：在全异构图训练中，对一对一属性使用BPR使属性嵌入余弦相似度高达0.93-0.98，但NDCG几乎不受影响，形成“静默故障”。
2. **CDL解决塌缩并提升属性区分性**：在所有数据集上CDL均显著提升属性线性探针AUC（+30-42个百分点）。
3. **排序改善取决于语义对齐**：在属性与偏好高度对齐的数据集上（Last.fm+7.8% NDCG, Yelp+2.9%, Audience Factory+3.3%），CDL同时提升排序；在低对齐数据集上（MovieLens-15.8%, BookCrossing-7.2%），CDL以可控排序代价换取向属性提升。
4. **梯度冲突存在且可被CDL缓解**：通过梯度余弦相似度测量（持续为负）确认BPR与CE在共享编码器上竞争，CDL通过联合训练导航此冲突。
5. **两个图属性决定CDL行为**：语义对齐（OTO属性是否预测偏好）和拓扑泄漏（图消息传递是否已隐含属性结构），可以用于预测CDL在特定数据集上的表现，无需运行完整实验。
