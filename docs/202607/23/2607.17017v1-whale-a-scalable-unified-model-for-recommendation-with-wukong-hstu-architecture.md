## 摘要
随着可扩展性在推荐建模中日益重要，最近的架构沿着两条独立的路径推进了两大类排序信号的建模：非序列特征（包括用户、物品、上下文和交叉特征）以及来自用户行为历史的序列特征。Wukong和HSTU已成为这些路径的代表性可扩展骨干网络：Wukong扩展了高阶非序列特征交互建模，而HSTU扩展了长用户行为序列建模。尽管它们具有互补优势，但结合这两种特征建模的实用架构仍未得到充分探索。我们提出了WHALE，一种可扩展的统一推荐架构，在Wukong和HSTU之上联合建模非序列和序列特征。每个WHALE层包含一个Wukong模块、一个HSTU模块和一个基于注意力的融合模块，其中Wukong派生的交互表示查询HSTU派生的行为表示。这种设计使两个骨干网络在整个网络中保持活跃，并实现渐进式的Wukong-HSTU交换，使高阶特征交叉能够反复从长用户历史中检索细粒度证据。为使WHALE适用于工业部署，我们引入了定制的Triton内核以及其他模型-系统协同设计技术，以提高训练和推理效率。在大规模工业推荐数据上，WHALE在离线实验中取得了持续改进。此外，它以适度的服务吞吐量折衷实现了正向在线收益。该方法已部署在生产系统中。总体而言，WHALE提供了如何在工业推荐模型中可扩展地统一这两类信息源的一个实际示例。

## Abstract
As scalability becomes increasingly important in recommendation modeling, recent architectures have advanced the modeling of two broad sources of ranking signals along separate paths: non-sequence features, including user, item, context, and cross features; and sequence features from user behavior histories. Wukong and HSTU have emerged as representative scalable backbones for these paths: Wukong scales high-order non-sequence feature-interaction modeling, while HSTU scales long user-behavior sequence modeling. Despite their complementary strengths, practical architectures that combine these two types of feature modeling remain underexplored. We present WHALE, a scalable unified recommendation architecture that jointly models non-sequence and sequence features on top of Wukong and HSTU. Each WHALE layer contains a Wukong module, an HSTU module, and an attention-based fusion module in which Wukong-derived interaction representations query HSTU-derived behavior representations. This design keeps both backbones active throughout the network and enables progressive Wukong-HSTU exchange, allowing high-order feature crosses to repeatedly retrieve fine-grained evidence from long user histories. To make WHALE practical for industrial deployment, we introduce customized Triton kernels and other model-systems co-design techniques to improve training and inference efficiency. On large-scale industrial recommendation data, WHALE achieves consistent gains in offline experiments. Additionally, it delivers positive online gains with a modest serving-throughput trade-off. The method has been deployed in production systems. Overall, WHALE provides a practical example of how these two sources of information can be scalably unified in an industrial recommendation model.

- **背景**：大规模推荐系统依赖两类关键信号：**非序列特征**（用户属性、物品属性、上下文、显式交叉）的高阶交互建模，以及**用户行为序列**（历史交互）的时序演化建模。现有代表性可扩展骨干网络分别为Wukong（侧重非序列特征交互）和HSTU（侧重序列建模），两者具有互补优势但通常沿着独立路径发展。
- **核心问题**：缺乏一种可扩展的架构，能够在同一个模型中**联合建模并渐进式交换**这两类信息。浅层混合方式（如先用序列模型压缩序列为摘要，再输入特征交互模型）限制了高阶交叉从长历史中检索细粒度证据的能力。
- **整体含义**：WHALE旨在通过每层的Wukong-HSTU交换，让静态特征交叉与动态序列模式相互增强，从而提升推荐质量与可扩展性。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- 构建递归堆叠的**WHALE层**，每一层同时包含**Wukong模块**（非序列特征高阶交互）、**HSTU模块**（序列建模）和**注意力融合模块**。Wukong输出作为查询（Query），HSTU输出作为键值（Key-Value），通过交叉注意力实现候选/上下文相关的细粒度序列证据检索，并将融合结果传递到下一层。
- 保持两个骨干网络在全网络活跃，实现**渐进式跨分支交换**。

### 2.2 输入层
- 非序列特征：分类字段通过嵌入表映射，数值字段通过MLP（数值特征分词器）映射，融合后得到 \(E_{ns} \in \mathbb{R}^{N \times D}\)。
- 序列特征：每个历史行为（物品ID+侧信息）通过嵌入层和MLP融合得到 \(E_s \in \mathbb{R}^{L \times D}\)。

### 2.3 WHALE层内部结构
- **Wukong模块**：应用堆叠的特征交互变换（FMB+LCB），输出 \(H_w^{(\ell)} \in \mathbb{R}^{N \times D}\)。
- **HSTU模块**：应用序列转导变换（自注意力+相对位置/时间偏置），输出 \(H_s^{(\ell)} \in \mathbb{R}^{L \times D}\)。
- **融合模块**：
  - 预层归一化后，Wukong查询 \(Q^{(\ell)}\) 与HSTU键值 \(K^{(\ell)}, V^{(\ell)}\) 计算交叉注意力：
    \[
    A^{(\ell)} = \text{softmax}\left( \frac{Q^{(\ell)} K^{(\ell)\top}}{\sqrt{D}} \right) V^{(\ell)}
    \]
  - 然后将 \(A^{(\ell)}\) 与 \(H_w^{(\ell)}\) 拼接，通过Fusion MLP（投影矩阵 \(W_F\)）和残差连接得到 \(\tilde{A}^{(\ell)}\)。
  - 再经SwiGLU FFN（共享门控变体）进一步变换，输出作为下一层非序列输入 \(X_{ns}^{(\ell+1)}\)。
- 序列分支直接传递 \(X_s^{(\ell+1)} = H_s^{(\ell)}\)。

### 2.4 效率优化
- **训练优化**：
  - 定制Triton注意力核：共享KV（减少GPU内存流量）；形状感知的异步反向变体（Q-parallel与KV-parallel选择），约1.2x加速。
  - 共享门控SwiGLU FFN：将门控和上投影矩阵绑定，减少33%矩阵乘法。
  - 混合精度训练（BF16骨干 + FP32数值敏感部分）和TorchInductor编译器优化。
- **推理优化**：
  - 形状感知Triton核调优、AOTInductor预编译算子融合（提升18%吞吐）。
