## 摘要
尽管大型语言模型（LLMs）因其强大的推理能力和广泛的世界知识最近在推荐系统中受到关注，但以往基于LLM的智能体存在幻觉和上下文长度限制，因此不适用于全排序推荐任务。为了通过架构设计而非修改LLM本身来规避这些限制，我们提出了一种基于智能体的推荐框架——基于记忆的自主语言代理个性化推荐工具学习（PRTA），其中LLM作为中央规划器，与多个推荐模型作为工具进行交互。基于LLM的智能体负责高层推理和个性化工具选择，而传统推荐模型利用其在建模行为模式方面的可扩展性，执行全排序评分。为了支持个性化工具选择，我们设计了反思机制，使智能体能够根据用户档案和候选排名列表评估和比较每个用户的工具。在三个公开数据集上的大量实验表明，PRTA在提高全排序推荐性能方面优于传统推荐和基于LLM的基线方法。

## Abstract
Although large language models (LLMs) have recently gained traction in recommender systems due to their strong reasoning capabilities and extensive world knowledge, previous LLM-based agents suffer from hallucination and context-length limitations, and thus are not suitable for full-ranking recommendation tasks. To circumvent these limitations through architectural design rather than modifying the LLM itself, we propose an agent-based recommendation framework, memory-based $\textbf{P}$ersonalized $\textbf{R}$ecommendation $\textbf{T}$ool learning via autonomous language $\textbf{A}$gents (PRTA), in which an LLM acts as a central planner interacting with multiple recommendation models as tools. The LLM-based agent is responsible for high-level reasoning and personalized tool selection, while traditional recommendation models perform full-ranking scoring, leveraging their scalability in modeling behavioral patterns. To support personalized tool selection, we design reflection mechanisms that enable the agent to evaluate and compare tools for each user based on user profiles and candidate ranked lists. Extensive experiments across three public datasets demonstrate the superiority of \modelname over traditional recommendation and LLM-based baselines in improving full-ranking recommendation performance.

# 论文总结：通过自主语言代理进行个性化推荐工具学习

## 1. 核心问题与整体含义（研究动机与背景）
- **核心问题**：大型语言模型（LLMs）在推荐系统中虽具有强大的推理能力和世界知识，但直接作为推荐器存在两个关键缺陷：**幻觉**（生成不存在的物品或无法映射到有效ID的描述）以及**上下文长度限制**（无法对完整的物品目录进行全排序）。因此，基于LLM的代理方法不适合实际的全排序推荐任务。
- **研究动机**：现有LLM代理方法通过精心设计提示词或者依赖外部验证工具来缓解幻觉，但效果依赖于LLM严格执行指令的能力；而全排序任务受限于LLM的token限制。论文主张**通过架构设计而非修改LLM本身**来规避这些限制。
- **整体含义**：提出一种将行为建模与语义推理解耦的模块化框架——**PRTA**，让LLM作为中央规划器，利用传统的推荐模型作为工具执行全排序，将LLM的推理能力专注于工具选择和轻量级重排序，从而既利用了传统模型的可扩展性，又发挥了LLM的语义理解优势。

## 2. 方法论
- **核心思想**：LLM代理负责高层推理和个性化工具选择，传统推荐模型负责全排序评分。每个用户都有一个可学习的**个性化工具记忆（PT Memory）** 向量，存储各工具的权重，通过加权聚合各工具的得分产生最终排名。
- **关键组件与流程**：
  1. **用户档案生成**：利用LLM根据用户历史交互物品的文本描述生成用户概要（user profile）。
  2. **推荐工具集**：三个预训练的推荐模型作为工具：LightGCN（图模型）、SASRec（序列模型）、SimpleX（矩阵分解模型）。它们输出全物品集的评分列表。
  3. **个性化工具记忆（PT Memory）**：一个与用户对应的权重向量（维度=工具数），初始值通过训练学习。最终的排名列表为各工具评分列表的加权和：$\hat{r} = m_G \hat{r}_G + m_S \hat{r}_S + m_M \hat{r}_M$。
  4. **代理反思机制（Agent Reflection）**：通过三种方式更新PT内存：
     - **局部工具评估**：LLM判断每个工具排名靠前的物品与用户近期行为的相关性（相关/中性/不相关），映射为数值并加权更新对应工具的权重。
     - **全局工具比较**：LLM在多个工具中选择“最相关”的一个，更新该工具权重。
     - **排名比较**：基于工具对用户近期交互物品的排名位置，计算每个工具的归一化倒数排名贡献，更新权重。这一机制提供非LLM的客观信号，减少LLM文本判断的偏差。
  5. **推断与上下文重排序**：从加权聚合的排名列表中选取Top-k候选，再次利用LLM根据用户档案和最近交互物品的文本描述对这些候选进行重排序，输出最终排名列表。
- **公式与算法流程**（文字说明）：见上述步骤。关键公式为局部评估的权重更新 $m_t \leftarrow m_t + \alpha \cdot y_{eval}^t$，全局比较的 $m_t \leftarrow m_t + \beta \cdot y_{comp}^t$，以及排名比较的归一化倒数排名求和。所有更新仅在训练阶段进行，每用户独立优化。

## 3. 实验设计
- **数据集**：三个公开数据集：
  - **Amazon**（电影和CD类别）：1,054用户，1,839物品，35,492交互。
  - **Yelp**：4,303用户，3,429商家，258,486交互。
  - **Goodreads**：3,204用户，2,167书籍，246,034交互。
  - 每个数据集按时间顺序组织交互序列，采用**留一法**（leave-one-out）进行训练/测试划分。
- **基准评测场景**：**全排序推荐**，即对所有物品进行排名（Full-ranking evaluation）。评估指标：Recall@10/20 和 NDCG@10/20。
- **对比方法**：
  - 传统推荐模型：LightGCN、SASRec、SimpleX（作为自身工具）、ENMF、DiffRec、FEARec。
  - 纯语言基线：BM25（基于文本相似度）、LLMRank（零样本LLM排序器）。
  - 检索增强生成（RAG）基线：LightGCN_RAG、SASRec_RAG、SimpleX_RAG（用LLM对单工具Top-50重排序）。
  - 注意：论文排除了其他基于LLM代理的工作（如AgentCF, RecAgent等），因为它们无法进行全排序，比较不公平。

## 6. 主要结论与发现
1. **PRTA在所有数据集上一致取得最佳性能**，提升显著（如Amazon数据集上Recall@10提升60%，NDCG@10提升66.15%）。
2. **纯语言方法（BM25, LLMRank）在全排序中表现极差**，因为它们无法建模协同和序列行为信号。
3. **RAG基线未能持续优于其基础工具**，因为LLM对单个检索器的候选列表重排序可能受到偏见；而PRTA聚合多个工具后，提高了鲁棒性。
4. **上下文重排序在语义一致的数据集（Amazon）上提升最大**；局部工具评估在语义信息较弱的场景（Yelp）更为关键；全局工具比较贡献稳定但相对较小。
5. **SASRec权重普遍较高**，说明序列模型更能捕捉用户偏好；SimpleX权重偏低，表明MF方法语义一致性较差。

## 7. 优点
- **架构巧妙**：通过将全排序任务委派给传统推荐模型，从根本上避免了LLM的幻觉和上下文长度限制，无需微调或复杂的提示工程。
- **可解释性**：PT内存中的工具权重直观反映了每个工具对用户的合适程度。
- **模块化与灵活性**：可以轻松替换或增加其他推荐模型作为工具，框架不依赖特定工具。
- **高效设计**：PT内存独立优化，可并行扩展；推理速度快（<0.5秒/用户）。
- **实验严谨**：消融实验清晰量化各模块的贡献，可视化分析提供了深入理解。
