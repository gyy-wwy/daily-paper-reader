Abstract 

New-user cold-start is a critical bottleneck for e-commerce plat-forms: predicting user lifetime value (LTV) and conversion rate (CVR) for users with sparse interaction history. Two prior directions— LLM-based semantic augmentation and learning using privileged information (LUPI)—each face a key limitation. First, LLM augmen-tation produces unstructured rationales that are noisy and hard to operationalize in production. Second, naive student-teacher distilla-tion can be brittle due to an information gap between the privileged teacher and the sparse student; moreover, this gap is heterogeneous across users. We propose SemRaD, a Semantic Reasoning-aware Distillation framework addressing both limitations. First, a Struc-tured Semantic Reasoning Pipeline replaces free-form rationales with a structured schema built via a discover–curate–audit work-flow, producing per user a Densified Semantic Profile (consumed by the deployed student via a Semantic-Gated Encoder that focuses on the most informative dimensions) and a Hindsight Distillation Target reconciled from pre- and post-conversion reasoning (used only at training). Second, to bridge this gap and handle its hetero-geneity, a Hindsight-Aware Distillation Network transfers privileged knowledge via the hindsight target, with Distillation Experts improv-ing transfer under per-user variability. On a large-scale industrial dataset, SemRaD lifts +1.9% LTV (Gini) and +1.0% CVR (AUROC) over a production-grade base; a four-week online A/B at Keeta confirms +1.0% LTV / +0.43% CVR. SemRaD also matches the pro-duction system’s LTV using only 9% of the training data while improving CVR by 0.8%.

1 Introduction 

New-user cold-start is a critical challenge for e-commerce user ac-quisition (UA) and lifecycle management. Upon signup, platforms must make early lifetime value (LTV) and conversion rate (CVR) predictions for a large number of users with sparse interaction his-tory. For ad and recommendation systems, LTV and CVR are central signals for UA and lifecycle management; extreme sparsity there-fore induces high-variance estimates and unreliable ranking deci-sions [ 12 , 14 ]. Standard ID-based sequence models (e.g., DIN [ 25 ], SASRec [ 8]) depend on inductive biases learned from rich histories; when user traces contain only a few events, their predictions often degenerate toward population-level averages, leading to inefficient marketing spend and missed retention opportunities. Two prior directions have attempted to address this sparsity, each facing a key limitation: (i) LLM-based semantic augmen-tation , which injects external world knowledge to densify sparse histories [ 3, 21 ], and (ii) learning using privileged information (LUPI) , where a teacher can access post-conversion signals during training [11, 18]. 

First , LLM-based augmentation can infer high-level user intent (e.g., style preference, functional constraints, brand affinity, brows-ing depth) from minimal context and densify sparse cold-start traces [ 3 , 16 , 21 ], with reasoning-centric approaches further elicit-ing natural-language rationales [ 9 , 17 , 24 ]. However, in production pipelines, unstructured rationales are difficult to validate, monitor, and operationalize: small prompt rephrasings or market shifts can yield inconsistent explanations even for the same user log [ 6 , 15 ], complicating downstream encoding and ranking. methods distill privileged features (available only during train-ing) from a stronger teacher into a student that uses only online-available signals [ 22 , 23 ]. However, the information gap between teacher and student is often too large for naive transfer to be robust, leading to brittle distillation that fails to inherit the teacher’s ad-vantage precisely in the sparse regime that motivates the approach; moreover, this gap is heterogeneous across users—the size and pat-tern of missing signal varies from user to user (Section 6.2)—and a single shared distillation pathway averages over these regimes, underfitting exactly the users for whom privileged information is most informative. To address both limitations together, we propose SemRaD, a 

Semantic Reasoning-aware Distillation framework (Figure 2). Sem-RaD has two complementary components, each addressing one limitation. (i) A Structured Semantic Reasoning Pipeline re-places free-form LLM rationales with structured outputs from a schema built via a discover–curate–audit workflow: per user, it produces a Densified Semantic Profile that ships with the deployed student, and a Hindsight Distillation Target reconciled from pre- and post-conversion reasoning, used only at training. (ii) A Hindsight-Aware Distillation Network then transfers privileged knowledge from teacher to student via the hindsight target, with Distillation Ex-perts handling per-user variability. Beyond the cold-start LTV/CVR setting reported here, the framework is being adopted for addi-tional production tasks at Keeta—including CTR prediction, user simulation, and silent-user re-engagement—which share the same structure: a deployable model predicting from sparse signal while richer outcomes are available only at training. Our contributions are threefold: 

• Structured Semantic Reasoning Pipeline. We introduce an LLM-driven pipeline that replaces unstructured ratio-nales with structured outputs from a discover–curate–audit 

schema-design workflow, producing per user a Densified Semantic Profile (consumed by the deployed student via a Semantic-Gated Encoder that focuses on the most informa-tive dimensions) and a Hindsight Distillation Target recon-ciled from pre- and post-conversion reasoning. 

• Hindsight-Aware Distillation Network. We propose a distillation network that bridges the teacher–student infor-mation gap via the hindsight target and handles per-user variability via Distillation Experts .

• Generalization & Production Deployment at Keeta. We show SemRaD generalizes across tasks (LTV regression and CVR classification), student backbones, and profiler LLMs, then validate it in production: a four-week online A/B con-firms +1.0% LTV and +0.43% CVR, and SemRaD matches the production system’s LTV using only 9% of the training data. The framework is further being adopted for other Keeta production tasks (e.g., CTR, silent-user re-engagement). 

2 Related work 

Cold-Start Prediction. Standard CVR/LTV backbones [ 4, 12 , 20 , 25 ,26 ] excel in warm-start regimes but collapse toward population pri-ors under extreme sparsity. Classical remedies (input dropout [ 19 ], self-supervision) treat the symptom but neither densify sparse in-puts with external knowledge nor exploit privileged training-time outcomes—the two levers we target. 

LLM-Based Semantic Augmentation. LLMs have been used for recommendation via prompting [ 2 , 3 ], fine-tuning [ 1], and reasoning-centric methods [ 9 , 17 , 21 , 24 ], but typically yield free-form ratio-nales that are hard to validate or operationalize. We instead generate 

structured , schema-guided reasoning offline and encode it with a frozen embedder for serving efficiency. 

Privileged Information and Distillation. LUPI [ 11 , 18 ], realized via distillation [ 5, 13 ] and privileged-feature distillation (PFD [ 22 ], HAPFD [ 23 ]), transfers privileged signals through a single shared pathway that is brittle under a heterogeneous gap. We instead add a reconciled hindsight target Z𝑓 𝑢𝑠𝑒𝑑 and per-user expert routing [ 7]. Extended discussion is in the Supplementary Material (§ S5).

4 Structured Semantic Reasoning Pipeline 

We address the cold-start information deficit by employing Large Language Models (LLMs) to project sparse user histories into a dense, interpretable semantic space (Figure 3). Our pipeline in-volves three LLM calls under a structured schema: (i) profiling S𝑝𝑟𝑒 

to generate the Densified Semantic Profile Z𝑝𝑟𝑒 (available at infer-ence), (ii) profiling S𝑝𝑜𝑠𝑡 to generate the privileged future profile 

Z𝑝𝑜𝑠𝑡 (training only), and (iii) fusing Z𝑝𝑟𝑒 and Z𝑝𝑜𝑠𝑡 to produce the 

Hindsight Distillation Target Z𝑓 𝑢𝑠𝑒𝑑 (training only). 

4.1 Stage 0: Schema Construction via Discover–Curate–Audit 

As discussed in the introduction, unstructured LLM-generated ratio-nales are difficult to validate, monitor, and operationalize in produc-tion. We therefore introduce a task-specific behavioral dimension schema D = {𝐷 1, . . . , 𝐷 𝐾 } that structures the LLM’s reasoning along predefined, interpretable axes (Figure 3, Stage 0), where each dimension 𝐷 𝑘 captures a distinct aspect of user behavior identified as predictive of conversion (e.g., Temporal Resolution , Transaction Magnitude , Navigation Intent Specificity , Domain Entropy ). Con-straining the LLM to these axes makes the reasoning (i) interpretable 

and verifiable against business intuition, (ii) consistent across users and time, and (iii) task-relevant rather than arbitrary narrative. 

Schema construction. Hand-engineering this schema is brittle, while ad-hoc LLM discovery risks irrelevant or unstable dimensions. We therefore adopt a three-step discover–curate–audit workflow that combines LLM-driven discovery, light human oversight, and a built-in empirical validation step. 

Discover. We stratify the training corpus into 𝐵 ≈ 100 batches of 

𝑛 =20 users each, balanced across activity strata (cold/warm/active/heavy) and outcome (high/low LTV). For each batch, an LLM proposes 3–6 candidate dimensions that discriminate the users and appear pre-dictive of LTV/CVR, yielding ∼500 raw candidates with substantial overlap. 

Curate. Candidates are deduplicated via embedding clustering (cosine ≥ 0.85 ) into 30–60 unique clusters, then consolidated by a second LLM call into 6–8 ranked finalists. Domain experts review these against a rubric checking distinctness, predictive relevance, and naming clarity, requiring domain familiarity rather than ML expertise; the deployed schema D contains 𝐾 =4 dimensions. 

Audit. After training, the mean gate weight ¯𝑤 gate ,𝑘 from the Semantic-Gated Encoder serves as an empirical utility score for each dimen-sion; consistently down-weighted dimensions are candidates for pruning. Section 6.2 reports per-dimension utilization for the de-ployed schema. Discovery and consolidation prompts, and dimen-sion examples, are in the Supplementary Material (§ S4) 1.

5 Hindsight-Aware Distillation Network 

The proposed SemRaD framework follows a Simultaneous Dis-tillation paradigm [ 22 ]. We instantiate two identical networks with shared architecture: a Student S and a Teacher T . They differ only in their inputs: S observes S𝑝𝑟𝑒 , while T observes both the regular and privileged data S𝑝𝑟𝑒 ⊕ S𝑝𝑜𝑠𝑡 . Both models are trained jointly. Crucially, our framework is Model-Agnostic . It is designed as a plug-and-play enhancement that can be integrated with any standard industrial backbone (DIN or Transformer). 

5.1 Multi-View Feature Encoding 

The Student and Teacher share encoder architectures across three views and differ only in inputs. The Static view embeds categori-cal and numerical user-profile features (shop ID, brand ID, tenure, session counts) via embedding lookups + standardized numerics, projected by an MLP to h𝑠𝑡𝑎𝑡 ∈ R𝑑 𝑠𝑡𝑎𝑡 ; both networks share this view since no privileged signal exists at the profile level. The Dy-namic view encodes the event sequence (event type, shop type, shop AOI; AOV, price, time gap) with a Transformer followed by attention-pooling:

The Semantic Reasoning view consumes the rationales from Sec-tion 4: the Student receives Z𝑝𝑟𝑒 and the Teacher receives the recon-ciled Z𝑓 𝑢𝑠𝑒𝑑 , both encoded by the Semantic-Gated Encoder below to produce h𝑆 𝑠𝑒𝑚 and h𝑇 𝑠𝑒𝑚 . 5.3 Distillation Experts 

Prior latent-alignment approaches (e.g., HAPFD [ 23 ]) transfer priv-ileged knowledge through a single shared projection pathway . This assumes that one student-to-teacher mapping is sufficient for all users, even though the teacher–student information gap can vary across users: for some users, post-conversion evidence may clarify pre-conversion ambiguity, while for others it may introduce signals that are difficult to infer from the student’s inputs. Table 1 summarizes the main comparison. Structured semantic descriptions improve over the base and unstructured augmentation baselines, while the full SemRaD model achieves the best results on both tasks: +1.9% LTV Gini and +1.0% CVR AUROC over Base. The lower block gives a compact component check: removing semantic gating or using only one distillation loss weakens the full recipe, although the KD-vs.-expert differences are close to seed variance. We therefore use the table as the primary quantitative evidence and focus the remaining main-paper analysis on the two learned mechanisms that make the method interpretable. 

Schema Utility via Gate Weights. The learned gate weights of the Semantic-Gated Encoder provide a post-hoc empirical utility score for each schema dimension (Section 4.1)—the Audit step of the discover–curate–audit workflow. Figure 5 shows that the gate is strongly dimension-selective—Navigation Intent Specificity dom-inates across all user segments while Transaction Magnitude is nearly suppressed—and that this ordering matches expert intuition. No dimension is consistently down-weighted enough to warrant pruning, confirming the schema is both interpretable and well-chosen.

6.3 Generalization and Robustness 

RQ3. Does SemRaD generalize across tasks, backbones, and profiler LLMs? SemRaD generalizes along two concrete axes (Ta-ble 2). First, replacing the Transformer sequence encoder with Avg-Pooling or DIN preserves the improvement, with the attention-based DIN benefiting most. Second, gains remain under alternative profiler LLMs, indicating that the effect comes from the structured semantic schema and hindsight fusion rather than a single upstream generator. 

6.4 Production Deployment 

RQ4. Is SemRaD deployable—data-efficient and effective in production? Data Efficiency. LTV labels are expensive and slow to collect, so we first ask whether structured reasoning can reduce dependence on large-scale supervision. In a small-data setting, Sem-RaD reaches production-level LTV quality using only 9% of the training data while improving CVR by 0.8% (Figure 1C), suggest-ing that the semantic profiles provide useful signal for new markets and other cold-start regimes.

7 Conclusion 

We presented SemRaD, a Semantic Reasoning–aware Distillation framework for new-user cold-start prediction in e-commerce. SemRaD tackles sparsity by (i) using an LLM to densify pre-conversion 

logs into structured semantic profiles, and (ii) distilling post-conversion 

privileged signals into teacher targets that are transferable to an online student constrained to pre-conversion inputs. Experiments on a large-scale industrial dataset show consistent gains on both LTV and CVR tasks, and strong data efficiency : using only 9% of the training data, SemRaD reaches nearly the same LTV (within 1%) while improving CVR by 0.8%. A four-week online A/B test at Keeta confirms these gains in production ( +1.0% LTV, +0.43% Conference acronym ’XX, June 03–05, 2018, Woodstock, NY Le et al. 

CVR). Finally, the proposed framework is model-agnostic and can be integrated with common industrial backbones.

S1 Implementation and Training Details S1.1 Model Architecture 

We implement SemRaD using PyTorch with the following configu-rations: 

• Static Encoder: A DNN based architecture containing a CrossNet (4 layers) and Deep Residual Blocks (2 blocks ×

2 layers, hidden dim=1024) to capture feature interactions. The output dimension is 𝑑 𝑠𝑡𝑎𝑡 = 128 .

• Dynamic Encoder: A Transformer Encoder with 2 layers, 4 attention heads, and a feed-forward dimension of 256. We employ Attention Pooling to compress the sequence into 

𝑑 𝑑𝑦𝑛 = 128 .

• Semantic-Gated Encoder: We utilize Qwen-0.6B-Emb to generate initial 1024-dimensional embeddings for each of the 𝐾 reasoning fields. These are processed by field-specific MLPs (Linear-BN-GELU) and projected to 𝑑 𝑎𝑢𝑔 = 512 .

• Multimodal Fusion: The three views are concatenated (128 + 128 + 512 = 768 ) and fused via a projection MLP to a shared representation of size 128. 

• Prediction Heads: Two separate DeepTaskHeads (3-layer MLPs) are used for LTV (regression) and CVR (sigmoid clas-sification). 

S1.2 LLM Protocols 

For semantic data generation, we utilize the open-source model 

GPT-OSS 120B as the inference engine for both student densifi-cation and teacher hindsight generation. For the text embedding layer in the Semantic-Gated Encoder, we use the pre-trained Qwen-0.6B-Embedding model frozen during training. 

S1.3 Training Configuration 

We train the model for 50 epochs with a batch size of 64 using the AdamW optimizer. The base learning rate is set to 10 −3 accom-panied by a 5-epoch linear warmup and cosine annealing decay. Weight decay is set to 5 × 10 −4 for regularization.

S5 Extended Related Work 

This section expands the discussion condensed in the main paper’s Related Work. 

S5.1 Cold-Start Prediction 

Classical collaborative-filtering and ID-based sequence models (DIN [ 25 ], DIEN [ 26 ], SASRec [ 8 ]) and feature-interaction models (DeepFM [ 4 ], DCN [ 20 ]) achieve strong warm-start accuracy but rely on dense in-teraction histories; under extreme sparsity their predictions regress toward population priors. Cold-start remedies fall into three fami-lies: (i) robustness methods such as DropoutNet [ 19 ] that randomly mask inputs to reduce over-reliance on dense features; (ii) meta-learning approaches that learn fast adaptation from few interac-tions; and (iii) self-supervised sequence objectives that pre-train representations. These mitigate the symptom of sparsity but do not inject external knowledge nor exploit privileged training-time outcomes—the two complementary levers SemRaD targets. 

S5.2 LLMs for Recommendation and Reasoning 

LLMs have been applied to recommendation through prompting and in-context formulations [ 2, 3], instruction tuning [ 1 ], and reasoning-centric pipelines that elicit natural-language rationales [ 9 ,17 , 21 , 24 ]. A recurring limitation is that free-form generations are sensitive to prompt phrasing [ 15 ] and prone to hallucination [ 6], which complicates validation and monitoring in production. Sem-RaD constrains generation to a curated behavioral schema, yielding structured, consistent rationales that are encoded by a frozen em-bedder at serving time so no LLM call is needed online. 

S5.3 Privileged Information, Distillation, and Mixtures of Experts 

Learning using privileged information (LUPI) [ 11 , 18 ] formalizes training-time-only signals, and is commonly realized via knowl-edge distillation [ 5 , 13 ]. In industrial recommendation, privileged-feature distillation (PFD [ 22 ]) and its hardness-aware extension (HAPFD [ 23 ]) feed post-conversion features to a teacher, but trans-fer through a single shared pathway. SemRaD differs in two ways: (i) it reconciles pre- and post-conversion reasoning into a self-consistent hindsight target Z𝑓 𝑢𝑠𝑒𝑑 rather than concatenating con-tradictory signals, and (ii) it routes transfer through per-user Distil-lation Experts [ 7], a mixture-of-experts mechanism that allocates reconstruction capacity to heterogeneous transfer regimes.
